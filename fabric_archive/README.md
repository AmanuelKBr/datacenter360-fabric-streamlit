# Fabric Artifacts Archive

This folder preserves the source artifacts of the **DataCenter360** Microsoft Fabric workspace
before the trial capacity expired. The Fabric trial does not support Git integration, Power BI
embedding, or offline `.pbix` export, so each item below was preserved manually through the
Fabric UI and committed here as plain files (notebooks, pipeline JSON, semantic model docs,
MLflow run details, report exports).

Combined with `app.py` and the gold-layer CSVs already in `/data`, this archive is enough to:
- Rebuild the Fabric workspace from scratch (notebooks, pipeline logic, semantic model)
- Demonstrate the full implementation to portfolio reviewers without a live Fabric workspace

## Checklist

- [x] Notebooks (4) → `notebooks/`
- [x] Pipelines (2) → `pipelines/` (full ARM template JSON + summary docs)
- [x] Semantic model → `semantic_model/` (tables/columns, no relationships, no DAX measures)
- [x] Operations Report → `reports/` (`.pbix`, live OneLake connection — see note below)
- [x] ML experiment results → `ml_experiment/` (MLflow run details for Isolation Forest)
- [ ] Lakehouse schemas (bronze/silver) → `lakehouse_schemas/` (templates only — not exported)
- [ ] Lakehouse bronze/silver data samples → `lakehouse_data/` (not exported)
- [ ] Executive Dashboard export → `reports/`

> The gold layer (`lh_gold`) does **not** need separate schema/data export — the CSVs in
> [`/data`](../data) are the gold table contents and are treated as the source of truth for the
> gold schema (no Dataflow Gen2 was used; all transformations happen in `nb_transform_gold`).
> Bronze/silver schemas and samples were not exported due to time constraints — these can be
> regenerated by re-running `pl_ingest_gsheets_bronze` / `nb_ingest_weather_bronze` and the
> transform notebooks against fresh source data if the project is rebuilt.

## 6. Lakehouse Schemas & Data (partially complete)

The **gold layer is fully preserved** as CSVs in [`/data`](../data) — no action needed for
`lh_gold`.

`lakehouse_schemas/lh_bronze_schema.sql` and `lh_silver_schema.sql` remain as templates (not
filled in). If revisiting this project with Fabric access, run on each table's SQL analytics
endpoint:

```sql
-- right-click table → "Script table as" → CREATE
```

and paste the DDL into the corresponding file. `lakehouse_data/bronze/` and `lakehouse_data/silver/`
are placeholders for `SELECT TOP 100 * FROM <table>` exports if needed in the future.
